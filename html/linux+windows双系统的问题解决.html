    linux+windows双系统的问题解决

    发布时间 : 2020-09-12 21:37
    字数:322 阅读 :
    1. 引导丢失
    2. 时钟变慢
    3. 共享蓝牙设备
      1. 0x01 linux连接设备
      2. 0x02 windows连接设备
      3. 0x03 修改linux文件

    引导丢失

    Windows更新后有时把Linux的引导EFI删了，BIOS中也看不到选项，可以在Linux启动盘下用efibootmgr新建一个

    efibootmgr -c -w -L "Kali Linux" -d /dev/sda -p 1 -l \\EFI\\kali\\grubx64.efi

    时钟变慢

    timedatectl set-local-rtc 1 --adjust-system-clock

    共享蓝牙设备

    双系统切换系统后另一个系统蓝牙键盘连接不上，只能重新配对
    原因是两个系统蓝牙设备配对成功后保存的key不同
    解决办法就是让两个系统的key保持一致

    0x01 linux连接设备

    打开bluetoothctl

    # 扫描蓝牙设备，查看设备Mac地址
scan on

# 获得Mac地址后关闭扫描
scan off

# 配对设备
pair 34:88:5D:98:5B:8E

# 连接设备
connect 34:88:5D:98:5B:8E

# 将设备添加到信任
trust 34:88:5D:98:5B:8E

# 查看已连接设备
devices

    0x02 windows连接设备

    配对后

    HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\BTHPORT\Parameters\Keys\电脑蓝牙MAC地址

    P.S. 貌似不能复制

    0x03 修改linux文件

    debian系的linux保存的Key一般在这个文件/var/lib/bluetooth/电脑蓝牙MAC地址/设备蓝牙MAC地址/info

    vi /var/lib/bluetooth/00:DB:DF:6B:F0:FC/34:88:5D:98:5B:8E/info

    将[LinkKey]下的Key值替换成windows注册表下的那个二机制值
